# Repository Port Contracts: Player State

## PlayerRepository

```python
from abc import ABC, abstractmethod

from ..entities.player_state import Player


class PlayerRepository(ABC):
    """CRUD operations on player identity/profile.

    The Player entity is the identity anchor for all player-related
    data. Other aggregates (PlayerState, future billing, stats)
    reference Player via player_id.
    """

    @abstractmethod
    async def create(self, player: Player) -> Player:
        """Create a new player.

        The player_id must be unique. Raises ValueError if a player
        with the same player_id already exists.
        Returns the created player.
        """
        ...

    @abstractmethod
    async def get(self, player_id: str) -> Player | None:
        """Get a player by ID. Returns None if not found."""
        ...

    @abstractmethod
    async def list(self) -> list[Player]:
        """List all players."""
        ...

    @abstractmethod
    async def delete(self, player_id: str) -> bool:
        """Delete a player by ID.

        Returns True if the player was deleted, False if not found.
        Does NOT cascade-delete associated PlayerState snapshots.
        """
        ...
```

### Contract rules

- `create()` raises `ValueError` if `player_id` already exists
- `get()` returns `None` if the player does not exist (never raises)
- `list()` returns an empty list if no players exist (never `None`)
- `delete()` returns `False` for non-existent players (never raises)
- `delete()` does not cascade to `PlayerState` snapshots — that is an application-layer decision
- All methods are `async`

## CardDefinitionRepository

```python
from abc import ABC, abstractmethod

from ..entities.card import (
    GeneralCardDefinition,
    HeroCardDefinition,
    RelicCardDefinition,
    TroopCardDefinition,
)


class CardDefinitionRepository(ABC):
    """Read-only access to the card definition catalog.

    Design-time port — card definitions are authored in Forge's
    configuration tools. Consumers of this port only read definitions.
    """

    @abstractmethod
    async def get_troop_definition(
        self, definition_id: str
    ) -> TroopCardDefinition | None: ...

    @abstractmethod
    async def list_troop_definitions(self) -> list[TroopCardDefinition]: ...

    @abstractmethod
    async def get_hero_definition(
        self, definition_id: str
    ) -> HeroCardDefinition | None: ...

    @abstractmethod
    async def list_hero_definitions(self) -> list[HeroCardDefinition]: ...

    @abstractmethod
    async def get_general_definition(
        self, definition_id: str
    ) -> GeneralCardDefinition | None: ...

    @abstractmethod
    async def list_general_definitions(self) -> list[GeneralCardDefinition]: ...

    @abstractmethod
    async def get_relic_definition(
        self, definition_id: str
    ) -> RelicCardDefinition | None: ...

    @abstractmethod
    async def list_relic_definitions(self) -> list[RelicCardDefinition]: ...
```

### Contract rules

- All methods are `async`
- `get_*` returns `None` if the definition does not exist (never raises)
- `list_*` returns an empty list if no definitions exist (never `None`)
- Implementations may include `save_*` convenience methods for testing/seeding, but these are not part of the port contract

## PlayerStateRepository

```python
from abc import ABC, abstractmethod

from ..entities.player_state import PlayerState


class PlayerStateRepository(ABC):
    """Append-only versioned persistence for player state.

    Play-time port — state is generated by players interacting with
    the game. Each save creates a new version snapshot.
    """

    @abstractmethod
    async def save(self, state: PlayerState) -> PlayerState:
        """Save a new version of the player state.

        The repository assigns the next version number and sets
        `timestamp` to the current UTC time automatically.
        The `version` field on the input state is ignored.
        The caller may set `change_note` to describe what caused
        this version (e.g., "Card troop-01 gained 50 XP").
        Returns the state with the assigned version and timestamp.
        """
        ...

    @abstractmethod
    async def load(
        self, player_id: str, version: int | None = None
    ) -> PlayerState | None:
        """Load a player's state.

        If version is None, returns the latest version.
        If version is specified, returns that exact snapshot.
        Returns None if the player or version does not exist.
        """
        ...

    @abstractmethod
    async def list_versions(self, player_id: str) -> list[int]:
        """List all available version numbers for a player, in ascending order."""
        ...
```

### Contract rules

- `save()` is append-only — it never overwrites or deletes existing versions
- `save()` assigns the version number; the caller's `state.version` is ignored
- `save()` sets `timestamp` to the current UTC time
- `save()` preserves the caller's `change_note` if provided
- `load(player_id)` without version returns the latest version
- `load(player_id, version=N)` returns exactly version N, or `None`
- `list_versions(player_id)` returns `[]` for non-existent players
- All methods are `async`
