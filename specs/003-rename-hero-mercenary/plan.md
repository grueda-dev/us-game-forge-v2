# Implementation Plan: Rename Hero Cards to Mercenary Cards

**Branch**: `003-rename-hero-mercenary` | **Date**: 2026-04-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-rename-hero-mercenary/spec.md`

## Summary

Rename all references to the "hero" card type to "mercenary" across the entire monorepo — enums, entities, ports, adapters, tests, documentation, and frontend. This is a mechanical refactor with zero behavioral changes. The `CardType.HERO` enum value becomes `CardType.MERCENARY`, all `Hero*` classes become `Mercenary*`, and all method/field names containing "hero" are updated to "mercenary".

**Primary Dependencies**: Angular, FastAPI, SQLModel, Pydantic, DuckDB-WASM
**Storage**: SQLite/PostgreSQL (Python), DuckDB-WASM (browser) — no data migration needed (development-stage project)
**Testing**: Jest (TypeScript), pytest (Python)
**Target Platform**: Web (Angular SPA + FastAPI backend)
**Project Type**: Monorepo (web-service + design tool)
**Performance Goals**: N/A (rename only)
**Constraints**: Existing Alembic migration files MUST NOT be modified
**Scale/Scope**: ~30 source files affected across 4 packages + 1 app + 3 doc files
